# 瀑布曲线卡片工具重构设计文档

- 日期：2026-04-13
- 项目：`waterfallTool`
- 目标平台：Blender 内的程序化瀑布卡片生成工具
- 第一版定位：从空物体发射一条可编辑水流轨迹，并沿轨迹生成可预览、可烘焙的 X 型连续交叉带状网格

## 1. 目标与定位

本次重构的目标不是继续做完整瀑布流体模拟器，也不是生成地形或多层瀑布系统，而是先做一个更聚焦、更可控的“曲线驱动瀑布卡片工具”。

第一版围绕单条水流轨迹工作。用户在 Blender 场景中放置一个空物体作为发射器，工具根据空物体位置、方向和速度参数生成一条水流轨迹。轨迹会和场景可见网格发生交互，默认表现为贴附滑流，并在满足脱离条件时继续下落。生成后的轨迹会转换为可编辑曲线，用户既可以手动调整形状，也可以选择让系统继续进行物理修正。最终，工具沿这条曲线生成一组连续的 X 型交叉带状面片，并在 bake 时输出正式静态网格。

## 2. 第一版范围

### 2.1 要解决的问题

第一版需要稳定支持以下能力：

- 从一个空物体生成一条主水流轨迹。
- 空物体位置作为发射点，空物体本地轴作为初速度方向，`Speed` 参数控制初速度大小。
- 关键参数保存到对象属性中，保证关闭并重开 `.blend` 文件后不丢失。
- 轨迹和场景可见网格发生交互，默认以贴附滑流为主，必要时脱离下落。
- 自动模拟结果转换为 Blender 曲线，用户可以手动调整曲线。
- 支持“手动定形”和“继续物理修正”两种曲线编辑模式。
- 沿曲线生成连续 X 型交叉带状网格，两片带子默认 90 度交叉，交叉夹角可调。
- 预览网格可实时响应曲线、宽度、交叉夹角和密度参数变化。
- 沿曲线方向的网格分段支持基础密度和曲率自适应加密。
- 宽度由用户设置起点宽度、终点宽度和宽度衰减，不在第一版强绑定速度。
- UV 沿流向连续展开，并通过辅助 UV 通道打包速度，主要用于驱动材质的 V 方向滚动速度。
- Bake 后生成独立正式网格，保留发射器和曲线，并自动关闭该曲线的预览。

### 2.2 暂不纳入第一版

为了保持重构范围清晰，第一版明确不做以下内容：

- 单个发射器自动分叉成多条轨迹。
- 多发射器批处理。
- 地形生成器。
- 喷溅、雾气、泡沫卡片等附加特效。
- 自动创建复杂材质节点。
- 自动 LOD 系统。
- 全局大场景性能保护或自动降级策略。
- 将 bake 结果转换成非 X 型的连续水体模型。

第一版仍会在求解器内部保留最大步数或最大模拟时长，作为停止条件，避免无限循环。这个限制只用于保证求解有终点，不作为大场景性能保护功能。

## 3. 用户工作流

第一版推荐工作流如下：

1. 用户在 Blender 中创建或选择一个空物体作为 `Emitter`。
2. 用户调整空物体位置和朝向，设置 `Speed` 等发射参数。
3. 工具根据空物体位置、方向、速度、重力、阻力和场景网格交互生成一条轨迹。
4. 工具把模拟轨迹转换为 `Flow Curve`。
5. 用户可以直接编辑曲线，也可以选择让系统基于曲线继续做物理修正。
6. 工具根据 `Flow Curve` 实时生成预览 X 型交叉带状网格。
7. 用户调整预览开关、基础分段密度、曲率加密强度、宽度参数和交叉夹角。
8. 用户确认后执行 bake。
9. 工具生成独立正式网格，保留 `Emitter` 和 `Flow Curve`，并自动关闭该曲线的预览。

## 4. 核心对象

### 4.1 Emitter

`Emitter` 是一个 Blender 空物体，负责定义水流发射状态。

`Emitter` 的位置作为轨迹起点。第一版默认使用空物体的本地 `-Z` 轴作为初速度方向，并在属性中通过 `direction_axis` 暴露为可选轴。默认 `-Z` 的原因是空物体不旋转时就能形成向下水流，用户需要横向喷射时可以旋转空物体或切换方向轴。用户本轮设计中确认的核心交互是“通过空物体朝向控制方向，通过 `Speed` 控制速度大小”，不需要第一版默认暴露 `Velocity X/Y/Z` 向量输入。

`Emitter` 至少保存以下持久化参数：

- `speed`
- `direction_axis`
- `gravity`
- `drag`
- `simulation_step_count`
- `simulation_time_step`
- `attach_strength`
- `detach_threshold`
- `preview_enabled`
- 对应 `Flow Curve` 的对象引用或名称

### 4.2 Flow Curve

`Flow Curve` 是工具的核心可编辑资产。它承载自动模拟结果，也承载用户手动调整后的轨迹。

`Flow Curve` 至少保存以下持久化参数：

- `curve_mode`：`Manual Shape` 或 `Physics Assisted`
- `preview_enabled`
- `base_segment_density`
- `curvature_refine_strength`
- `start_width`
- `end_width`
- `width_falloff`
- `cross_angle`
- `uv_speed_scale`
- 原始模拟速度缓存
- 对应 `Emitter` 的对象引用或名称

曲线在手动编辑后不会丢失水流语义。若处于 `Manual Shape` 模式，预览网格完全跟随用户编辑后的曲线。若处于 `Physics Assisted` 模式，系统会基于当前曲线重新做一次贴附与脱离修正，再更新曲线和预览网格。

### 4.3 Preview Mesh

`Preview Mesh` 是实时显示用对象，由 `Flow Curve` 和几何参数生成。它不是最终交付资产，可随参数和曲线变化反复重建。

预览网格必须和 bake 网格使用同一套几何和 UV 规则，避免预览结果与最终结果脱节。

### 4.4 Baked Mesh

`Baked Mesh` 是执行 bake 后得到的正式静态网格。它复制当前预览生成逻辑的结果，并写入正式 `UV0` 和速度辅助 UV 通道。Bake 不删除 `Emitter` 和 `Flow Curve`。Bake 完成后，该 `Flow Curve` 的预览开关自动关闭，避免正式网格和预览网格重叠显示。

## 5. 轨迹模拟与场景交互

### 5.1 初始状态

轨迹初始状态来自 `Emitter`：

- 起点为 `Emitter` 的世界坐标位置。
- 初速度方向来自 `Emitter` 的指定本地轴方向。
- 初速度大小来自 `speed`。
- 初始速度向量为方向乘以速度大小。

### 5.2 基础受力

第一版求解器包含重力和可选阻力：

- 重力用于形成下落趋势。
- 阻力用于减缓速度，避免轨迹过度抛射化。
- 时间步和最大步数由参数控制。

### 5.3 场景交互

第一版默认检测场景中可见网格对象，而不是要求用户指定单独碰撞集合。工具自身生成的 `Preview Mesh` 和 `Baked Mesh` 默认不参与碰撞检测，避免轨迹和自己的显示网格发生自碰撞。

碰撞后的默认行为不是强反弹，而是贴附滑流：

- 当轨迹命中表面且满足贴附条件时，速度方向投影到表面切向，轨迹沿表面滑动。
- 当表面支撑不足、法线变化过大、速度方向不再适合贴附或脱离阈值被满足时，轨迹脱离表面，继续受重力影响下落。
- 第一版可以保留很弱的偏折参数，但不把“像抛射物一样反弹”作为主要表现。

### 5.4 曲线编辑模式

第一版支持两种曲线编辑模式：

- `Manual Shape`：用户手动编辑曲线后，曲线形状完全以用户结果为准。系统只根据曲线重建预览网格，不自动改动曲线。
- `Physics Assisted`：用户手动编辑曲线后，系统基于当前曲线重新做一次碰撞、贴附和脱离修正，再重建预览网格。

## 6. X 型交叉带状网格

### 6.1 网格形态

第一版生成的是沿 `Flow Curve` 延展的连续 X 型交叉带状面，而不是很多完全断开的独立小卡片。

每条轨迹生成两片连续带子：

- 两片带子共享同一条中心轨迹。
- 两片带子默认 90 度交叉。
- 用户可以通过 `cross_angle` 调整交叉夹角。
- Bake 结果保持 X 型交叉带状网格形态，不转换为其他连续水体模型。

### 6.2 分段密度

沿曲线方向的分段由两个因素共同决定：

- `base_segment_density` 控制整体分段密度。
- `curvature_refine_strength` 控制曲率自适应加密强度。

曲率越大的区域分段越密，曲率越小的区域分段越疏。这样可以减少弯曲处折线感，也能降低直线段不必要的面数。

### 6.3 宽度控制

第一版宽度由用户控制，不强制绑定速度。

宽度参数包括：

- `start_width`
- `end_width`
- `width_falloff`

工具根据曲线归一化长度计算每个采样截面的宽度。默认配置可以保持全程固定宽度，也可以做从起点到终点逐渐收窄的水流形态。

### 6.4 稳定局部坐标系

每个采样截面需要基于曲线切线建立局部坐标系，用来放置两片交叉带子的左右边界点。为了避免曲线弯曲或竖直下落时面片突然翻转，第一版需要使用稳定的 frame 传播策略，而不是每个点都直接用世界上方向独立计算截面。

## 7. UV 与速度表现

第一版速度主要通过 UV 数据服务材质动画，而不是先做复杂顶点色或 Geometry Attribute。

UV 规则如下：

- `U` 方向横跨每片带子的宽度，从左边到右边映射到 `0` 到 `1`。
- `UV0` 的 `V` 方向沿水流轨迹累计长度展开。
- 轨迹求解时每个采样点保存速度值。
- 如果曲线被手动编辑，系统根据相邻点距离和原始速度缓存估算新的速度分布。
- `UV1` 或等价辅助 UV 通道存储归一化速度值，供材质驱动 `V` 方向滚动速度，使水流更快的位置滚动更快。

第一版不要求额外输出速度顶点色或 Geometry Attribute。速度作为 UV 数据打包给材质使用，避免把速度表现死绑在顶点色方案里。

## 8. 实时预览与 Bake

### 8.1 实时预览

第一版默认尽量实时刷新：

- 发射器参数变化后刷新轨迹和预览。
- 曲线编辑后刷新预览。
- 宽度、分段密度、曲率加密和交叉角变化后刷新预览。

第一版不做大场景性能保护或自动降级。用户选择的是优先实时反馈。求解器内部仍保留最大步数或最大模拟时长，保证模拟有确定终点。

### 8.2 Bake 行为

Bake 操作执行以下行为：

1. 读取当前 `Flow Curve` 和几何参数。
2. 使用和预览一致的几何生成规则生成正式网格。
3. 写入正式 `UV0` 和用于速度的辅助 UV 通道。
4. 创建独立 `Baked Mesh` 对象。
5. 保留 `Emitter` 和 `Flow Curve`。
6. 自动关闭该 `Flow Curve` 的预览开关。

Bake 不删除控制对象，也不把曲线转成不可逆状态。用户后续仍可以打开预览、继续调整曲线和参数，并再次 bake 出新的网格。

第一版 UI 以选中的 `Emitter` 或 `Flow Curve` 为入口。

建议分为以下区域：

## 10. 数据保存

第一版必须把关键设置保存到对象属性中，保证 `.blend` 文件重启后仍可恢复。

保存策略如下：

- 发射相关设置保存在 `Emitter` 上。
- 曲线、网格、预览和 bake 相关设置保存在 `Flow Curve` 上。
- 预览网格可重建，不作为核心状态来源。
- Bake 网格是正式输出资产，不反向驱动曲线。

## 11. 验收标准

第一版完成后需要满足以下标准：

- 从一个空物体能自动生成一条可编辑水流曲线。
- 空物体位置控制发射点，空物体朝向控制初速度方向，`Speed` 控制速度大小。
- 关键参数保存到对象属性中，保存并重开 `.blend` 后不丢失。
- 曲线能和场景可见网格交互，默认贴附滑流，必要时脱离下落。
- 用户可以在 `Manual Shape` 和 `Physics Assisted` 两种曲线编辑模式之间切换。
- 手动编辑曲线后，预览 X 型带状网格能实时更新。
- 调整基础密度、曲率加密、宽度和交叉夹角时，预览能实时变化。
- 曲率较大的区域能自动获得更高分段密度。
- UV 沿流向连续，辅助 UV 通道中的速度值能用于材质 V 方向滚动速度表现。
- Bake 后生成独立正式网格，保留控制对象和曲线，并自动关闭该曲线预览。

## 12. 风险与约束

第一版需要注意以下风险：

- 默认检测场景可见网格，在复杂场景中可能较慢。当前设计按用户选择先不做全局性能保护。
- 曲线手动编辑后，原始速度分布可能不再完全准确。第一版通过距离和原始速度缓存估算速度变化，后续可扩展更明确的速度控制。
- X 型交叉带在急弯处容易扭转，需要稳定 frame 传播策略。
- 贴附与脱离阈值如果过于敏感，轨迹可能在表面抖动，需要在测试中调默认值。
- 第一版不做自动分叉，因此复杂瀑布需要未来通过多发射器或后续分叉系统扩展。

## 13. 最终定义

第一版产品定义如下：

一个 Blender 内的曲线驱动瀑布卡片生成工具。它从单个空物体读取发射点、方向和速度，模拟一条能与场景可见网格交互的水流轨迹，把轨迹转换为可编辑曲线，并沿曲线生成可实时预览、可曲率自适应加密、可调宽度和交叉夹角的 X 型连续交叉带状网格。用户 bake 后得到独立正式网格，同时保留发射器和曲线用于后续迭代。
