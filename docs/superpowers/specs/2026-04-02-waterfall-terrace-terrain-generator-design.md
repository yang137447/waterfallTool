# Blender 台地瀑布地形生成器设计文档

- 日期：2026-04-02
- 项目：`waterfallTool`
- 目标平台：Blender 内的程序化地形生成模块，结果直接服务瀑布工具与中模展示

## 1. 目标与定位

本模块不是通用自然景观生成器，而是一个面向瀑布生产流程的核心台地地形生成器。

第一版目标是在 Blender 中生成可直接展示的双层或三层台地瀑布核心地形，并输出可编辑唇边、阻流块和推荐出水线，供现有瀑布生成工具继续使用。

## 2. 第一版范围

### 2.1 要解决的问题

第一版需要稳定支持以下能力：

- 生成双层或三层台地瀑布核心地形
- 用主轴控制整体结构，用参数和少量曲线控制每层唇边
- 自动生成每层默认唇边草案
- 自动生成缺口和切流岩块
- 输出连续主地形和独立阻流块
- 输出推荐出水线与可编辑唇边
- 支持局部替换某层唇边、增删缺口和岩块
- 支持和现有瀑布工具的基础联动

### 2.2 暂不纳入第一版

为了保持范围聚焦，第一版明确不做以下内容：

- 完整周边山体环境
- 大范围多瀑群景观
- 植被、表层散件和生态覆盖
- 高精侵蚀雕刻
- 实时双向联动瀑布重算
- 自动生成完整关卡级地貌

## 3. 用户工作流

第一版推荐工作流如下：

1. 艺术家创建整体主轴。
2. 设置层数、总落差、台地进深和宽度趋势。
3. 工具自动生成层级骨架、每层唇边、缺口、阻流块和推荐出水线。
4. 艺术家按层级局部替换某层唇边，或增删缺口与岩块。
5. 工具生成 `MainTerrain` 与 `BlockerMasses`。
6. 艺术家选择或调整推荐出水线。
7. 将主地形、阻流块和出水线交给现有瀑布工具做求解与重建。

## 4. 系统架构

模块分为四层：

### 4.1 结构层

结构层负责生成台地关系与跌落结构，不直接生成最终中模网格。

输出内容包括：

- 主轴
- 层级骨架
- 每层唇边草案
- 缺口区段
- 阻流岩块布局
- 推荐出水线

### 4.2 形体层

形体层负责将结构数据转为中模地形。

输出内容包括：

- `MainTerrain`
- `BlockerMasses`
- 每层台地面和跌落唇边的连续形体

### 4.3 交互层

交互层负责让艺术家对自动生成结果做局部接管。

支持内容包括：

- 替换某层唇边
- 增删缺口
- 增删阻流块
- 启用或忽略某条推荐出水线

### 4.4 联动层

联动层负责将地形生成结果输出给瀑布工具。

输出内容包括：

- 主地形碰撞体
- 独立阻流块碰撞体
- 可识别唇边曲线
- 推荐出水线对象

## 5. 技术路线

### 5.1 总体方案

采用以下混合方案：

- Python 负责层级结构、每层唇边草案、缺口、岩块和推荐线布局
- Geometry Nodes 负责将结构层数据生成可展示的中模地形
- Blender Addon 负责交互、覆盖编辑和与瀑布工具的对象级联动

### 5.2 方案原因

这样拆分有三点好处：

- 结构逻辑清晰，适合和现有瀑布工具共享数据
- Blender 中模形体仍保持可视化与参数化
- 后续若继续发展为更完整的景观系统，也不会被纯节点图绑死

## 6. 结构层数据模型

结构层建议维护以下核心对象。

### 6.1 Terrain Blueprint

总蓝图对象至少记录：

- 层数
- 主轴
- 总落差
- 整体宽度趋势
- 唇边圆润度
- 缺口频率
- 阻流块密度
- 默认随机种子

### 6.2 Terrace Level

每一层台地至少记录：

- `level_index`
- `elevation`
- `terrace_depth`
- `terrace_width`
- `drop_height_to_next`
- `basin_strength`
- `lip_profile_mode`

### 6.3 Lip Curve Draft

每层默认唇边草案至少记录：

- 所属层
- 曲线点列
- 曲率强度
- 连续区段
- 缺口区段
- 是否被人工替换

### 6.4 Gap Segment

每个缺口至少记录：

- 所属层
- 在唇边上的起止区段
- 缺口深度强度
- 是否贯穿到下一层
- 是否被人工锁定

### 6.5 Blocker Mass

每个阻流块至少记录：

- 所属层或层间区域
- 中心位置
- 宽度
- 高度
- 前突程度
- 与唇边的相对关系
- 自动生成还是手工放置

### 6.6 Suggested Emitter

每条推荐出水线至少记录：

- 来源层
- 曲线点列
- 推荐强度
- 是否默认启用
- 是否被用户替换

## 7. 自动生成逻辑

自动生成建议按以下顺序执行：

### 7.1 生成层级骨架

首先根据主轴和全局参数确定：

- 层数
- 每层高度
- 每层落差
- 每层进深
- 整体宽度变化趋势

### 7.2 生成每层默认唇边

每层默认唇边由以下因素共同决定：

- 主轴控制整体方向
- 每层自身的弧形或折线偏移
- 受随机种子影响的自然扰动

### 7.3 生成缺口

缺口由这些因素驱动：

- 唇边局部弯曲
- 台地宽度变化
- 层间落差大小
- 阻流块位置关系

### 7.4 生成阻流块

阻流块建议优先出现在：

- 中层承接区
- 层间过渡区
- 与缺口关系密切的位置

### 7.5 推导推荐出水线

推荐出水线来自仍然连续、适合跌落的唇边区段。缺口区和被阻流块完全打断的区段不作为主要推荐线。

## 8. 艺术家干预方式

### 8.1 全局参数级

艺术家可以直接调：

- 层数
- 每层落差
- 台地进深
- 整体宽度
- 唇边圆润度
- 缺口频率
- 阻流块密度

### 8.2 层级覆盖级

艺术家可以对某一层单独改：

- 唇边曲线
- 台地宽度和进深
- 缺口多少
- 连续性强弱
- 推荐出水线启用状态

### 8.3 局部编辑级

第一版只开放少量关键能力：

- 替换某一段唇边
- 新增或删除一个缺口
- 新增或删除一个阻流块
- 接受或忽略某条推荐出水线

### 8.4 编辑状态

为了避免系统覆盖人工调整，建议每个可编辑结构对象维护：

- `Auto`
- `Overridden`
- `Locked`

## 9. 输出与和瀑布工具的联动

### 9.1 主地形输出

- `MainTerrain`

作为连续主地形，同时也是瀑布工具的默认主碰撞体。

### 9.2 独立阻流输出

- `BlockerMasses`

作为展示几何和局部强分叉碰撞体。

### 9.3 唇边与推荐线输出

- 每层 `Lip Curves`
- `Suggested Emitters`

作为瀑布工具的推荐出水输入。

### 9.4 层级与辅助数据

建议保留：

- 每层 level id
- 每层所属区域标识
- lip 的编辑状态
- 推荐线的启用状态

## 10. 推荐联动工作流

推荐的跨模块工作流如下：

1. 用地形生成器生成 `MainTerrain`
2. 自动生成 `Lip Curves` 与 `Suggested Emitters`
3. 艺术家接受或微调推荐出水线
4. 现有瀑布工具读取推荐线作为出水线输入
5. `MainTerrain` 与 `BlockerMasses` 作为碰撞来源
6. 继续做瀑布求解与重建

## 11. 实现阶段拆分

### 阶段 A：结构原型

目标是快速生成层级骨架与默认唇边：

- 输入主轴
- 输入层数和整体参数
- 自动生成每层唇边
- 自动生成推荐出水线

### 阶段 B：中模成形

目标是将结构层转为可展示中模：

- 生成连续主地形
- 生成唇边厚度与圆润形体
- 生成独立阻流块

### 阶段 C：局部编辑

目标是提高镜头与艺术可控性：

- 替换某层唇边
- 增删缺口
- 增删阻流块
- 接受或忽略推荐线

### 阶段 D：和瀑布工具联动

目标是将生成结果接入现有瀑布生成器：

- 输出标准命名对象
- 现有瀑布工具直接读取推荐出水线
- 现有瀑布工具读取主地形和阻流块作为碰撞体

## 12. 验证标准

### 12.1 结构验证

- 层级关系清楚
- 每层唇边可识别
- 推荐出水线落在合理区段

### 12.2 造型验证

- 地形读起来像层叠台地而不是普通台阶
- 缺口和阻流块能清晰切流
- 至少形成两个以上可跌落区

### 12.3 联动验证

- 推荐出水线可直接喂给现有瀑布工具
- 主地形和阻流块可直接作为碰撞体使用
- 新地形 demo 驱动出的瀑布结果明显优于当前平崖壁示例

## 13. 风险与约束

实现过程中需要重点关注以下风险：

- 纯参数生成容易产生程序化台阶感
- 唇边和缺口关系不清时，推荐线质量会很差
- 阻流块数量过多会让地形失去主结构
- 中模成形如果过度平滑，层级关系会丢失
- 局部人工覆盖如果没有状态管理，容易被自动生成覆盖

第一版应通过明确结构对象、编辑状态和演示模板来控制这些风险。

## 14. 最终定义

第一版产品定义如下：

一个由 Python 结构生成、Geometry Nodes 中模成形、Blender Addon 负责交互的台地瀑布核心地形生成器。它面向瀑布搭建流程，重点解决台地层级、可跌落唇边、缺口、阻流岩块和推荐出水线的生成与编辑问题，并直接服务现有瀑布工具的后续求解与重建。
